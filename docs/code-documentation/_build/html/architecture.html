Architecture

High-level flow

Client / Demo / ADK
       │
       ▼
validation_workflow.run_validation_workflow()
       │
       ▼
workflow_engine.execute_workflow()
       │
       ├── CacheAgent ──► CapabilityStatement (TTL + ETag/304)
       ├── CapabilityInterpreterAgent
       ├── QueryValidatorAgent (+ pattern detection)
       ├── QueryExecutionAgent (if valid + mode allows)
       └── RuleAgent ──► SearchLearnerAgent | HumanInterventionGate

Optional upstream step: QueryGeneratorAgent populates query_url from standard FHIR search parameters when query_generation is supplied.

ADK graph wrapper

The Google ADK Workflow in src.agentic_layer.graph.validation_workflow is a thin graph over the shared engine:

START → initialize_workflow → run_validation_pipeline → finalize_output

Escalation (learner / human) runs inside src.agentic_layer.graph.workflow_engine.execute_workflow().

State model

src.agentic_layer.state.workflow_state.ValidationWorkflowState is the Pydantic model passed through ADK nodes and demo entry points.
